17 class Vehicle(object):
18  """The Vehicle class is the main entry point for the OpenXC Python library.
19  A Vehicle represents a connection to at least one vehicle data source and
20  zero or 1 vehicle controllers, which can accept commands to send back to the
21  vehicle. A Vehicle instance can have more than one data source (e.g. if the
22  computer using this library has a secondary GPS data source).
23 
24  Most applications will either request synchronous vehicle data measurements
25  using the ``get`` method or or with a callback function passed to
26  ``listen``.
27 
28  More advanced applications that want access to all raw vehicle data may want
29  to register a ``DataSink`` with a Vehicle.
30 
31  @author Christopher Peplin github@rhubarbtech.com
32  @date June 25, 2013
33  @version 0.9.4"""
